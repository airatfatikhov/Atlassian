# 🔧 Рассинхронизация задач в Jira на серверах: диагностика и исправление

Проблема рассинхронизации задач в Jira обычно возникает в кластерных окружениях (Data Center), когда разные узлы показывают разное состояние задач. Вот полное руководство по диагностике и исправлению.

# 🎯 Основные симптомы рассинхронизации

❌ Задача создана на узле А, но не видна в поиске на узле Б
❌ Разное количество задач в навигаторе на разных узлах
❌ Поисковые запросы возвращают разные результаты на разных нодах
❌ Гаджеты и дашборды показывают противоречивые данные
❌ Индекс репликации отстаёт на минуты/часы

# 📋 Быстрая диагностика
## 1. Проверьте Health Check в админке
Jira Administration → System → Cluster → Health Check

## 2. Проверьте индексы через REST API
````
    curl -u admin:password http://<node-url>/rest/api/2/index/summary | jq .
````

## 3. Проверьте синхронизацию времени
````
    # На каждом сервере
    date
    ntpq -p  # или chronyc tracking
````

# 🔍 Основные причины и решения
🕐 1. Рассинхронизация системного времени

Решение:
# Linux: настройка NTP
````
sudo timedatectl set-ntp true
sudo systemctl restart systemd-timesyncd
````

# Проверка
````
timedatectl status
````
